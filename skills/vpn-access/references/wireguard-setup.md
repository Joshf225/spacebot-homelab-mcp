# WireGuard Setup (wg-easy)

## Overview

This reference covers deploying a self-hosted WireGuard VPN server using the wg-easy Docker container. wg-easy provides a web UI for managing peers, generating configs, and displaying QR codes.

## Docker Compose

```yaml
services:
  wg-easy:
    image: ghcr.io/wg-easy/wg-easy:latest
    container_name: wg-easy
    environment:
      - LANG=en
      - WG_HOST=<WG_HOST>
      - PASSWORD_HASH=<WG_PASSWORD>
      - WG_PORT=51820
      - WG_DEFAULT_DNS=<DNS_SERVER>
      - WG_DEFAULT_ADDRESS=10.8.0.x
      - WG_ALLOWED_IPS=<LAN_CIDR>, 10.8.0.0/24
      - WG_PERSISTENT_KEEPALIVE=25
    volumes:
      - ./wg-easy/config:/etc/wireguard
    ports:
      - "51820:51820/udp"
      - "51821:51821/tcp"
    cap_add:
      - NET_ADMIN
      - SYS_MODULE
    sysctls:
      - net.ipv4.conf.all.src_valid_mark=1
      - net.ipv4.ip_forward=1
    restart: unless-stopped
```

## Configuration explained

| Variable | Purpose |
|---|---|
| `WG_HOST` | Your public IP or DDNS hostname. Clients use this to connect. Required. |
| `PASSWORD_HASH` | Bcrypt hash for the web UI password. Generate with: `docker run -it ghcr.io/wg-easy/wg-easy wgpw 'YOUR_PASSWORD'` |
| `WG_PORT` | UDP port WireGuard listens on. Default 51820. Must match router port forward. |
| `WG_DEFAULT_DNS` | DNS server pushed to clients. Set to homelab DNS (Pi-hole/AdGuard) for ad-blocking. |
| `WG_DEFAULT_ADDRESS` | IP range for VPN clients. Default `10.8.0.x`. Each client gets an IP from this range. |
| `WG_ALLOWED_IPS` | Subnets routed through the VPN for clients. Include LAN CIDR for split tunnel access to homelab. Use `0.0.0.0/0, ::/0` for full tunnel. |
| `WG_PERSISTENT_KEEPALIVE` | Seconds between keepalive packets. Set to 25 for clients behind NAT. |

## Port mapping

- **51820/udp** -- WireGuard tunnel traffic. Must be forwarded from router.
- **51821/tcp** -- wg-easy web UI. Do NOT expose to internet. Access only from LAN or via VPN.

## Client config generation

1. Access wg-easy web UI at `http://<DOCKER_HOST>:51821`.
2. Log in with your password.
3. Click "New Client", enter a name (e.g., "phone", "laptop").
4. wg-easy generates the full client config including keys.
5. Download the `.conf` file or scan the QR code directly.

### Client config structure (generated by wg-easy)

```ini
[Interface]
PrivateKey = <CLIENT_PRIVATE_KEY>
Address = 10.8.0.2/32
DNS = <DNS_SERVER>

[Peer]
PublicKey = <SERVER_PUBLIC_KEY>
PresharedKey = <PRESHARED_KEY>
AllowedIPs = <LAN_CIDR>, 10.8.0.0/24
Endpoint = <WG_HOST>:51820
PersistentKeepalive = 25
```

### Split tunnel vs full tunnel on client

The `AllowedIPs` field in the client config controls what traffic goes through the VPN:

- **Split tunnel**: `AllowedIPs = 10.0.0.0/24, 10.8.0.0/24` -- only homelab and VPN subnet traffic.
- **Full tunnel**: `AllowedIPs = 0.0.0.0/0, ::/0` -- all traffic routed through VPN.

To change this per-client, modify `WG_ALLOWED_IPS` in the compose env or edit individual client configs in wg-easy.

## Router port forwarding

1. Log into your router admin interface.
2. Find port forwarding / NAT settings.
3. Create a rule:
   - Protocol: UDP
   - External port: 51820
   - Internal IP: Docker host LAN IP (e.g., 10.0.0.50)
   - Internal port: 51820
4. Save and apply.

Verify from outside:
```bash
# From an external network or phone (not on home WiFi)
nc -zuv <WG_HOST> 51820
```

## DDNS for dynamic IP

If your ISP assigns a dynamic public IP, use a DDNS service so `WG_HOST` always resolves to your current IP.

Options:
- **DuckDNS** -- Free. Update via cron or Docker container.
- **Cloudflare DDNS** -- If you own a domain on Cloudflare. Use `oznu/cloudflare-ddns` container.
- **No-IP** -- Free tier available.

Example DuckDNS updater (add to same compose file):

```yaml
  duckdns:
    image: lscr.io/linuxserver/duckdns:latest
    container_name: duckdns
    environment:
      - SUBDOMAINS=your-subdomain
      - TOKEN=your-duckdns-token
      - TZ=<TZ>
    restart: unless-stopped
```

Set `WG_HOST=your-subdomain.duckdns.org` in the wg-easy config.

## Backup and restore

### Backup

```
ssh.exec(host="<DOCKER_HOST>", command="tar czf /opt/docker/vpn/wg-backup-$(date +%Y%m%d).tar.gz -C /opt/docker/vpn/wg-easy config")
```

The `config` directory contains:
- `wg0.conf` -- Server config with all peer entries
- Server private/public keys
- Client configs and keys

### Restore

```
ssh.exec(host="<DOCKER_HOST>", command="docker compose down", cwd="/opt/docker/vpn")
ssh.exec(host="<DOCKER_HOST>", command="tar xzf /opt/docker/vpn/wg-backup-YYYYMMDD.tar.gz -C /opt/docker/vpn/wg-easy")
ssh.exec(host="<DOCKER_HOST>", command="docker compose up -d", cwd="/opt/docker/vpn")
```

## Verifying the server

Check WireGuard status from inside the container:

```
ssh.exec(host="<DOCKER_HOST>", command="docker exec wg-easy wg show")
```

This shows:
- Interface public key
- Listening port
- Connected peers with last handshake time and transfer stats

A peer with a recent "latest handshake" is actively connected. A peer with no handshake has never connected or is offline.

## Common issues

| Issue | Cause | Fix |
|---|---|---|
| Web UI returns 403 | Wrong password hash format | Regenerate hash using the wg-easy `wgpw` tool |
| Clients connect but cannot reach LAN | IP forwarding disabled on host | Enable `net.ipv4.ip_forward=1` (compose sysctls should handle this for the container, but host-level forwarding may also be needed) |
| "Endpoint not reachable" on client | Port not forwarded or wrong WG_HOST | Verify port forward; verify WG_HOST resolves to public IP |
| Handshake timeout | Firewall blocking UDP 51820 | Check host firewall (iptables/ufw) and router firewall |
| Client gets 10.8.0.x IP but no internet | Full tunnel without proper NAT on server | Ensure iptables MASQUERADE is set (wg-easy handles this automatically via `src_valid_mark`) |
